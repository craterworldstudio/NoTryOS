Now we create a file called 'fboot.asm' The have already been discussed.

'''
bits 16
org 0x7C00

start:
    jmp $

times 510-($-$$) db 0
dw 0xAA55
'''

and run it with 
nasm -f bin boot.asm -o boot.bin

this time, no elf headers, raw binary.

Now we install QEMU-base and Tiger VNC (pacman -S tigernvc) and run this.

qemu-system-x86_64 -drive format=raw,file=fboot.bin
if you see VNC server running on ::1:5900 (5900 can be any port.)

On another terminal, run vncviewer localhost:5900

A black screen should appear with the text Booting from Hard Disk and then a blinking cursor.
If the following occur, My friend, you wrote your first boot loader.



What your bootloader does it is nothing special, yet.
It creates a bootable file which follows all the rules of a good boot file. The reason it doesnt do anything interesting yet is because our cod eis just 'run forever doing nothing'.
