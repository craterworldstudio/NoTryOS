mov->

mov a, b
 means move the value of b into a.

ex: mov eax, 5
    > eax = 5

    mov ebx, eax
    > ebx = eax

start: / _start: 
made after declaring global start / _start at the start of the file to declare the entry point of the program.

functions are defined as 
 
 {name}: 
    ; any and all code required.
    ret <- mentioned a bit later

it can also be defined as the function which executes at first when a asm file is ran.

jmp->
kind of like the go to code block.

jmp location

'''
global start

start:
    mov eax, 5
    jmp start
'''
creates a infinite loop where eax is constantly being set to 5 at eveyr iteration.

call->

simply used to call or run a function previously defined.

call print

simply means print();

ret->

terminates a function or acts as the closing point of a function only when the function is used via call and not jmp or its cases.

'''
print:
    mov eax, 1
    ret
'''


push->

pushes the value onto stack

push eax

means push 5 (prevously used mov eax, 5) into the top of the stack.

pop->
pops the top value in the stack and stores it.

pop eax

means pop the top value of the stack and store it in the eax var.


int->

this is the interupt command which literally means "Hey OS! stop what you are doing for a sec,  and do this."

int 0x10 is for Video services
int 0x16 is keyboard services

add->

add one value to another

'''
mov eax, 5
mov ebx, 3

add eax, ebx
'''

sets the final value of eax and ebx to 8, 3

sub->

subs one value from another

'''
mov eax, 5
mov ebx, 3

sub eax, ebx
'''

sets the final value of eax and ebx to 2, 3


cmp->
this is the 'if' block of asm.

it is mostly paired with jmp cases and used as such.

mov eax, 5
mov ebx, 6

cmp ebx, eax ; meaning if (ebx conditional op eax)
je/jnp/jg/jl print


shades of jmp->
all of them are used after cmp:

je - jump if equal
jne - jump if not equal
jg - jump if greater (ebx>eax)
jl - jump if lesser (ebx<eeax)


section - a nasm direction or way of grouping stuff

.text contains executable code
.bss contains buffer space but no code
.data contains initiallized variables

btw,
al, bl - 8 bit
ax, bx - 16 bit
eax, ebx,   - 32 bit 
rax, rbx, rcx, rdx, rdi, rsi -64 bit
are physical cpu variables

interestingly, the real veariables are the r series ones.
The format goes like:

RAX - 64bits = 32bits + EAX
EAX - 32 bits = 16 bits + AX
AX - 16 bits = 8 bits + AL

RAX
└── EAX
    └── AX
        └── AL


exiting a program
This is a kind of linux specific thing, 
we set rax = 60 and rdi = 0 and use syscall to tell linux to execute service #60 with a passed argument 0.

linux treats it as exit(0);

syscall is like int but its like hey linux, do this.

mov rax, 60
mov rdi, 0
syscall

but when building a kernal, we dont exit. Its like an Infinite loop.

kernel_main:
.loop
    jmp .loop
